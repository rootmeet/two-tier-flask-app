# two-tier-flask-app
🚀 Building a Resilient Flask App with MySQL, Docker, and Kubernetes🌐
 <br />Flask-based web application that seamlessly interacts with a MySQL database. Users can submit messages through the user-friendly interface, and these messages are securely stored in the MySQL database, then promptly displayed on the frontend.
 ![image](https://github.com/rootmeet/two-tier-flask-app/assets/145815260/a469ee6c-9e98-4c86-aab6-46f210a7c125)
Are you ready to take your web application to the next level in terms of scalability and resilience? Look no further!

🌐 Application Overview

We've developed a Flask-based web application that seamlessly interacts with a MySQL database. Users can submit messages through the user-friendly interface, and these messages are securely stored in the MySQL database, then promptly displayed on the frontend.

🐳 Docker Containerization

For streamlined packaging and deployment, we've containerized our application with Docker. This ensures consistent behavior across different environments and simplifies the deployment process.

🌟 Future-Proofing with Kubernetes

But that's not all! Our journey doesn't stop here. To create a robust and auto-scalable application that can effortlessly handle increased loads and recover from failures automatically, we're planning to migrate this application to a Kubernetes-based architecture.
